# AGENTS.md - City Council Workspace

## Every Session
1. Read `SOUL.md` — your role
2. Read `USER.md` — who you're helping
3. Read `memory/YYYY-MM-DD.md` (today + yesterday) for recent context
4. In main session, also read `MEMORY.md`

## Meeting Workflow

### Before a Meeting
When asked to prep for an upcoming meeting:
1. Read the agenda from `meetings/agendas/`
2. For each agenda item:
   - Identify if it involves an ordinance or legal question
   - Check `reference/preemption-notes.md` and `reference/oregon-revised-statutes.md`
   - Web search for current ORS sections if needed
   - Note any public comment topics or controversial items
3. Write a briefing doc to `meetings/prep/YYYY-MM-DD-prep.md`
4. Flag items needing legal review, votes, or decisions

### During / After a Meeting
When given meeting notes or minutes:
1. Extract all action items with owners and deadlines
2. Track motions and vote outcomes
3. Flag any ordinance readings (1st, 2nd, 3rd)
4. Note items tabled or continued to next meeting
5. Save to `meetings/minutes/YYYY-MM-DD-minutes.md`
6. Update `MEMORY.md` with significant decisions

### Tracking
- Open action items live in `meetings/action-items.md`
- Ordinance pipeline lives in `ordinances/pipeline.md`

## Ordinance Analysis Workflow

When asked to analyze a proposed ordinance:
1. Identify the subject area
2. Check `reference/preemption-notes.md` for known preemption
3. Web search for current ORS sections (oregonlegislature.gov)
4. Web search for relevant federal law
5. Check if other Oregon cities have similar ordinances
6. Write analysis to `ordinances/analysis/[topic]-[date].md`
7. Summarize findings with citations

## Memory
- Daily notes: `memory/YYYY-MM-DD.md`
- Long-term: `MEMORY.md` — decisions, legal interpretations, institutional knowledge
- Capture council votes, legal positions confirmed by city attorney, and policy directions
